📦 Package D: Calendar Page Issues

#14 — Fix tooltip/hover cutoff (Portal approach)

Problem: Tooltips inside calendar day squares get clipped by overflow-y-auto on the parent square.

Solution:

  • Create a Tooltip.svelte component that renders at document body level
  • On hover, calculate target element's bounding rect and position the tooltip absolutely to the viewport
  • Remove the inline tooltip HTML from calendar squares, replace with the portal component
  • Keep the same visual style (background, border, arrow indicator)

Alternatives considered:

  • CSS overflow-visible — simpler but layout shifts possible
  • Fixed-position tooltip — works everywhere but more event management

#12 — Remove scroll, clip/truncate tasks

Problem: Each day square has overflow-y-auto, causing messy scrollbars inside tiny squares.

Solution:

  • Remove overflow-y-auto class from day squares
  • Tasks beyond visible area are naturally clipped by the square's dimensions
  • Use truncate class on task titles to prevent text overflow
  • Full task list still accessible via clicking the day → DayDetailModal

#10 — Make calendar page mobile-friendly

Problem: Calendar grid doesn't adapt well to small screens.

Solution (sensible defaults based on the "all of the above" answer):

  • Cramped grid: Keep 7-column grid (calendar needs it), reduce padding/gap on small screens. Already has gap-px on mobile.
  • Touch targets: Increase day square tap areas by keeping aspect-square with larger effective touch zone
  • Task text unreadable: On very small screens (< 380px), hide task titles inside squares entirely — show only a colored dot/badge per task. Details available via tap → DayDetailModal.
  • Month nav: Already responsive with order-last on mobile — the month nav wraps below the title, which is good.
  • Back button: Already there (and will use BackButton component from Package C).
